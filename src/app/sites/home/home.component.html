Angular guards - canLoad

To present authentication and authorization in real world example, some modules are guarded by angular guards.

For example Coin API module is guarded by identifyGuard (canLoad) guard.
To get access, You need to log in and get JWT (json web token) from backend identify/authorisation server. Token is set in every request to server to verify user on server-side. Preventing nonAuthorised access to API resources

  • userName: testowo@gmail.com
  • password: 123456

Session time expires after 2 minutes. After valid login, every 1sec checks token validity. Token expires makes auto logout. Need to logIn again

user-info if loggedIn is also saved to localStorage (removed if expired/loggedOut)
